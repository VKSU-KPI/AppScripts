# General Election Automation Scripts

This folder contains the core Google Apps Script files used to manage the interactive dashboard for election tracking. These scripts work together to ensure data integrity, prevent user errors, and automate the creation of standardized election documents based on predefined templates.

### `election_faculty_menu.gs`

This script is responsible for dynamically updating the user interface of the dashboard. It runs automatically in the background using the built-in `onEdit` trigger every time a user modifies a cell in the spreadsheet. The script specifically monitors cell `E2`, which acts as the primary control for selecting the election type.

We use this background monitoring approach to keep the dashboard context-aware and prevent invalid data entry before it happens. When the user selects an election type from the predefined array of types that require a faculty division, the script dynamically generates a data validation rule. It reads the available faculties from the range `A2:A24`, flattens this data into a single-dimensional array, and filters out any empty rows. This ensures the resulting dropdown menu in cell `E4` is always clean and accurate. If the chosen election type does not require a faculty, the script explicitly clears the data validation and the content of cell `E4`. We do this to ensure users cannot accidentally assign a faculty to a university-wide election, which would break the logic of the reporting structure.

### `election_sheet.gs`

This script handles the automated generation of new spreadsheet tabs and enforces our document naming conventions. Unlike the menu script, this function is designed to be triggered manually by the user, typically by clicking a designated drawing or button on the dashboard sheet. It reads the final configuration from cells `E2` and `E4` (election type & faculty) to determine what kind of document needs to be created.

The core purpose of this script is to eliminate manual copying and ensure every new election sheet is perfectly formatted. We implemented strict validation checks right at the beginning of the function to stop the execution and show UI alerts if the user forgets to select a required field. This prevents the creation of broken or incomplete sheets. The script determines the correct template by automatically appending the word "ШАБЛОН" to the selected election type. We chose this strict naming convention to avoid writing complex mapping logic; the script simply trusts that a corresponding template exists. It also constructs the expected name of the new sheet and checks if it already exists to prevent duplicate entries. After successfully duplicating the template and renaming it, the script forcibly clears the dashboard inputs and removes the faculty dropdown. We added this final step to reset the dashboard state completely, guaranteeing that the next operation starts with a clean slate and no residual data from the previous run.
